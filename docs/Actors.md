# Actors and Responsibilities

## 1) Staff

- Submit transport requests
- Provide destination, department, preferred date, passengers, budget
- Attach supporting scheduling PDF (optional)
- Track request status and history
- Export personal request/trip PDF reports

## 2) Timetabling Staff

- Receive staff requests as source inputs
- Validate request details and requested date
- Schedule each request into timetable entries
- Submit timetable entries to manager as source-of-truth set
- Export timetable PDF

## 3) Transport Manager

- View pending queues and submitted timetables
- Approve/reject/confirm requests where needed
- Allocate buses and drivers
- Use soft override where conflicts exist
- Manage driver records
- Export operational/manager reports

## 4) Driver

- View assigned trips
- Record trip start and end times
- Submit incident/mechanical/delay/fuel/other notes
- Export driver report PDF

## 5) Dean

- Oversight role for club-related trip approvals and visibility
- Export dean report PDF

## 6) System Admin Behavior (Preassigned Accounts)

- Certain roles (for example manager/timetabling) are pre-seeded accounts
- Staff can additionally self-register via signup flow
